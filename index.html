DSharp Model Viewer

Tip: Select your DSharp project root folder. The viewer understands either: - DSharp exports in a _Content folder (Model/Submodel/Class/Association mementos), or - Simple model.json files and classes/*.json per model. References can be class IDs or paths like "Model/SubModel/ClassName". To load from Azure DevOps Git, set AZDO_PAT on the server and fill Org, Project, Repo, Branch.

Assumptions: Each model is a directory; optional model.json with {name, classes, submodels}. Classes may also be in classes/*.json. A class belongs to one model but may be referenced elsewhere; such visiting classes are shown with dashed borders.
